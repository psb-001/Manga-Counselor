## Contributing Guidelines

Thanks for your interest in contributing! To keep changes easy to review:

- Use small, focused pull requests.
- Follow the existing code style (TypeScript, React, Tailwind).
- Run `npm run build` before submitting to ensure the project compiles.
- Prefer descriptive commit messages (e.g., `feat: add reading progress card`).

### Development
1. Fork and clone the repo
2. Install deps: `npm install`
3. Start dev server: `npm run dev`
4. Build: `npm run build`

### Code Style
- Type-safe components and hooks
- Meaningful naming and clear props
- Avoid adding unused files or dead code

### Reporting Issues
Open an issue with a minimal reproduction or clear description.
